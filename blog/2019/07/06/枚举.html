数字枚举

假设后台定义三个状态：

1 待支付 2 待发货 3 已完成

前端使用的时候，如果整个项目中到处都是1、2、3, 有个问题就是不够语义，维护起来时间成本也较高， 想办法映射成变量：

enum Status {
  outPay = 1,
  outDelivery = 2,
  done = 3
}

当使用都时候，可以判断Status.outPay === status即可

针对数字枚举，还有一个反向映射， 即Status[1]值是outPay
